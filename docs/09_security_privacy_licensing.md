# Security, privacy, and licensing

## Project privacy

- User schematics stay local by default.
- No schematic upload without consent.
- Offline mode is available.
- A report of what data was sent to the LLM.
- Optional masking of component / net names.

## API keys

- Keys stored locally.
- Encryption in a later phase.
- Corporate BYOK.
- Cost limits per project / user.

## Source licensing

- Public sources are not automatically open-source.
- Use links, metadata, summaries, and our own rules.
- Do not redistribute full vendor content as our own database.
- Do not ingest paid standards without a license.

## LTspice

- Do not distribute LTspice.
- Do not host LTspice as a service.
- Do not imply an official partnership without consent.
- The user configures their own installation.

## Vendor models

- Every model carries its own license.
- Do not bundle without checking the terms.
- In the MVP, use simple substitute models and paths to the user's local libraries.
