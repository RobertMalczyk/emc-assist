## What this golden tells you when it changes

If a future pipeline run produces materially different output from these files:
- **DM and CM both non-negligible**, with CM much larger than expected → chassis capacitance model was added (planned for M2.10) or a regression in dual-LISN wiring.
- **Different LTspice numbers from baseline** → look at `composer.py` (LISN/cable instantiation), `netlist/fragment.py` (ground rename), `results/raw_parser.py` for regressions.
- **Different recommendation count or content** → look at the prompt template (`prompts/recommendations_v1.md`), the retrieval scoring (`knowledge/retrieve.py`), or the LLM-model defaults.
- **New Rule_ID citations not in seed jsonl** → the LLM is hallucinating; check the prompt's "no fake citations" guardrail.
- **Verbatim vendor text or URLs appearing in the privacy log prompt** → the redaction layer is broken (`feedback_copyright_redaction_for_llm` rule violated); investigate `redact_for_llm`.

## How to regenerate (not overwrite)

The pipeline writes new outputs to `generated/`, `results/`, and `reports/` (all gitignored). It never writes here. To refresh the golden after an intentional behaviour change:

```powershell
# 1. Run the pipeline as documented above.
# 2. Copy the fresh outputs over the old golden:
Copy-Item .\reports\report.md .\golden\report.md -Force
Copy-Item .\generated\recommendations.json .\golden\recommendations.json -Force
Copy-Item .\results\llm\rep-*.jsonl .\golden\llm_privacy_log.jsonl -Force
# 3. Update the "Date" / "Commit" / "Cost" / "Key results" fields in this README.
# 4. Commit the refresh with a clear message explaining what changed and why.
```

## Caveats

- LLM output is non-deterministic. Re-running with the same model and prompt will produce *similar* but not identical text. Don't diff for exact equality; check for structural / semantic regressions.
- Cost is approximate; OpenAI may adjust pricing.
- This snapshot uses `--llm-mode replace` and the default dual-LISN topology. A single-LISN snapshot (legacy mode) or `augment` snapshot would be separate files.

## Historical note — M2.7 era (single-LISN)

The earlier (M2.7) version of this snapshot used a single positive-rail LISN. Its V(MEAS) peak was 3.41 V and the conducted-band peak was 72.1 dBµV. The numbers differ from the current dual-LISN snapshot because the single-LISN measurement mixed CM and DM contributions into one rail. The dual-LISN version cleanly separates them — CM ≈ 0 reflects that no chassis-coupling path is modeled, which is the right answer for this simulation as-built.
